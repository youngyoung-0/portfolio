## 1. 문제 발생

프로젝트를 설정하던 중 예기치 않은 에러가 발생했습니다.

```bash
Error: command not found: hugo
```

## 2. 원인 분석

정적 사이트 생성기인 Hugo가 개발 환경에 설치되어 있지 않았기 때문입니다. 홈브루(Homebrew)를 통해 간단히 설치할 수 있었습니다.

## 3. 해결 방법

다음 명령어를 통해 Hugo를 전역으로 설치했습니다.

```bash
brew install hugo
```

이후 `hugo version` 명령어로 정상 설치 여부를 확인한 뒤, 무사히 프로젝트 생성을 마칠 수 있었습니다.

## 4. 배운 점

- 개발 환경을 세팅할 때는 필요한 CLI 도구가 로컬에 잘 설치되어 있는지 먼저 확인하는 습관을 들여야겠습니다.
- 환경 구성 시 발생하는 에러 메시지를 당황하지 않고 꼼꼼히 읽으면 원인을 쉽게 파악할 수 있음을 배웠습니다.
