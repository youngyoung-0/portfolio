## 프로젝트를 시작한 이유

페르소나는 많은 기업에서 구매 의사 예측, 세그먼테이션, 추천 시스템 등 분석 보조 도구로 활용되고 있습니다. 하지만 저희는 한 가지 의문이 들었습니다.

> LLM에 하나의 인격을 부여했을 때, 유저가 던지는 예측 불가능하고 복잡한 상황 속에서도 그 페르소나를 안정적으로 유지할 수 있을까?

이를 실험하기 위한 무대로 **게임**이 적합하다고 판단했습니다. 게임 환경은 돌발적인 대화를 자연스럽게 유도하고, 실시간 NPC 반응을 통해 페르소나의 일관성을 눈으로 직접 확인할 수 있기 때문입니다.

세계관으로는 **사이비 종교**를 선택했습니다. 외부와 단절된 폐쇄 환경에서 대화·심리에 집중할 수 있고, 비이성적인 가치관과 '생존과 탈출'이라는 명확한 목표가 유저의 몰입감을 극대화할 수 있다고 생각했습니다.

## 게임 소개

**UMI**는 사이비 종교를 배경으로 한 LLM 기반 인터랙티브 게임입니다.

- 고유한 성격, 말투, 신념을 가진 입체적인 NPC 캐릭터들과 실시간으로 대화합니다.
- NPC는 과거 대화를 기억하고, 현재 스탯(친밀도, 신뢰도, HP 등)에 맞춰 실시간으로 반응을 바꿉니다.
- 사전 분류 모델이 대화 의도를 12가지 태그로 파악하여 스탯 증감 및 생존 분기에 즉각적으로 반영합니다.

### 플레이 사이클

- 유저는 맵을 이동하며 탐색을 통해 단서와 아이템을 획득합니다.
- NPC와 대화하며 허점을 공략하거나 친밀도를 쌓습니다.
- 특정 아이템(솔피의 눈물)을 사용하면 오염도가 상승해 어안 렌즈 같은 패널티가 생기지만, 고위 NPC와 대화할 수 있는 기회를 얻습니다.
- 하루가 끝나면 그날 발생한 사건을 Story Agent가 요약하고 장기 기억으로 동기화합니다.
- 5일차 종료 후, 누적된 요약으로 엔딩이 생성됩니다.

![플레이 사이클](/assets/images/umi-game-page/umi_play_cycle.png)

## 기술 스택

| 역할 | 기술 |
|---|---|
| API 서버 | **FastAPI** + Uvicorn (ASGI 비동기) |
| 데이터베이스 | **MongoDB** (Motor 비동기 드라이버) |
| LLM 오케스트레이션 | **LangChain**, LangSmith (모니터링·트레이싱) |
| 모델 추론 | **HuggingFace Transformers**, BitsAndBytes (양자화), PEFT |
| 임베딩 & 벡터 DB | **Sentence-Transformers**, Vector DB (RAG) |
| 인증 | **JWT** |
| 인프라 | **Docker**, Nginx |
| 데이터 검증 | **Pydantic** |

## AI 에이전트 구조

### NPC Agent — Gemma-2-9b-it

모델 선정 과정에서 Llama, Mistral, Gemma를 비교 평가했습니다. 응답 속도와 한국어 성능에서 **Gemma**가 가장 우수했습니다.

초기에는 PEFT 기법을 적용했으나, 세계관을 이탈하는 현상이 빈번히 발생했습니다. 이후 **SFT(Supervised Fine-Tuning)** 으로 전환하여 페르소나 일관성을 확보했습니다. 구어체 자연스러움을 위해 GPT-4o mini로 학습 데이터를 생성한 뒤 **Gemma-2-9b-it** 모델을 파인튜닝했습니다.

![gemma-2-9b-it](/assets/images/umi-game-page/gemma-2-9b-it.png)

### Guardrail — 2단계 안전장치

유저의 발화가 NPC Agent에 전달되기 전에 두 단계의 가드레일을 통과합니다.

**(Safety):** `bert-base-multilingual-cased` 모델을 사용해 욕설·비속어·탈옥 시도를 감지합니다. AI 허브의 '텍스트 윤리 검증' 데이터셋으로 파인튜닝했습니다. 3회 감지 시 인 게임 하루를 강제로 넘기는 패널티가 부여됩니다.

![bert-base-multilingual-cased](/assets/images/umi-game-page/bert-base-multilingual-cased.png)

### Story Agent — Qwen-7b

Llama, Mistral, Gemma를 스토리 요약 작업으로 비교했을 때, **Qwen**이 어휘력과 표현력에서 가장 뛰어났습니다.

Story Agent는 NPC별 대화 내용을 요약해 **Vector DB**에 장기 기억으로 저장합니다. 이 기억은 이후 RAG 방식으로 검색되어 NPC의 답변 생성에 활용됩니다.

![qwen-7b](/assets/images/umi-game-page/qwen-7b.jpg)

## 전체 파이프라인

유저가 발화하면 다음 흐름으로 처리됩니다.

1. **GA1 Safety** → 욕설·탈옥 감지
2. **분류 모델** → 12가지 의도 태그 파악
3. **NPC Agent** → 스탯 반영 후 답변 생성 (대화 마스킹 적용)
4. **Story Agent** → 대화 요약 후 Vector DB에 저장

![AI 에이전트 흐름도](/assets/images/umi-game-page/ai-agent-flow.png)

### 백엔드 안정성 확보

- **비동기 파이프라인:** BERT부터 Qwen까지 이어지는 다중 에이전트 구조를 `asyncio` 기반으로 비동기 처리하여 병목을 최소화했습니다.
- **출력 포맷 안정화:** LLM의 비정형 출력을 Pydantic 스키마로 변환하여 런타임 오류를 원천 차단했습니다.
- **리소스 모니터링:** CPU·RAM·GPU 사용량을 실시간으로 확인할 수 있는 헬스 체크 엔드포인트를 구현했습니다.

## 회고

### 페르소나를 만든다는 것

페르소나 구현이 단순히 설정한 성격대로 말하게 하는 것이 아님을 뼈저리게 느꼈다. 어떤 어투로, 어떤 상황에서, 어떤 감정으로 말하는지까지 인문학적으로 고민하지 않으면 진짜 '인격체'를 만들 수 없다는 걸 깨달았다. 기획에 2주 이상을 쏟은 이유이기도 했고, 그게 동시에 개발 시간을 압박한 이유이기도 했다.

### RAG의 현실적인 한계

게임 내에서 LLM과 대화를 거듭하면서 **Vector DB RAG의 구조적 한계**를 체감했다. 단순 키워드 유사도로 메모리를 검색하다 보니, 문맥과 무관한 기억을 불러오거나 인물 간의 관계를 혼동하는 문제가 반복됐다. 이 문제를 해결하기 위한 방향으로 **Graph RAG**를 알게 됐다. 대화 내역을 엔티티와 관계 기반의 노드로 구조화하면, '누가 누구를 의심하는지' 같은 고차원적인 관계 추론이 가능해진다. 시간이 됐다면 꼭 적용해보고 싶었다.

### 응답 지연 문제

페르소나를 입힌 LLM과의 실시간 대화에서 **1.3~2.5초** 정도의 응답 지연이 발생했다. 문장 수를 제한해서 어느 정도 줄였지만, 게임의 몰입감을 높이려면 더 줄여야 한다고 느꼈다. 이후 vLLM의 **PagedAttention**과 **Prefix Caching** 기법을 찾아봤는데, NPC 페르소나와 금기어 프롬프트처럼 매 턴 반복되는 고정 입력을 캐싱 처리하면 추론 부하를 크게 줄일 수 있다는 걸 알게 됐다. 5주라는 시간 안에 기획-개발-배포-발표를 모두 처리해야 했기에 적용하지 못한 게 아쉬웠다.

## 고도화 로드맵

- **Graph RAG 도입:** Vector DB 기반 RAG를 대체하여 인물 관계망 기반의 정확한 기억 검색 구현
- **vLLM + PagedAttention:** 메모리 단편화 해결 및 첫 토큰 생성 시간(TTFT) 단축
- **Prefix Caching:** 반복 입력되는 고정 프롬프트 캐싱으로 추론 부하 절감
